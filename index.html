Accessible Forms

Communicate clearly

Don't Ask for unneccessary or confusing data

Be device-friendly

Communicate Clearly: Error Handling

Don't show validation errors while typing

Don't show validation errors after form submission

Do show validation errors on blur

Be clear with what's required (or what isn't)

Be upfront about format requirements

Communicate Clearly: Options

Don't put a two-option input in a dropdown

If there are only a few options, use a radio group

Bacon Sausage

Collapse groups of options into option-groups to save label space

Communicating Clearly: Labels vs. Placeholders

Don't use placeholders as labels

Use placeholders to show sample input

Web Workers and Service Workers


// index.js
var worker = new Worker('worker.js');
worker.postMessage('ping');
worker.onmessage = function(e) {
  console.log(e.data);   // 'pong'
};
            

// worker.js
self.onmessage = function(e) {
  console.log(e.data); // 'ping'
  self.postMessage('pong');

};
            

// index.js
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.register(
    '/service-worker.js');

}
          

// service-worker.js
self.addEventListener('fetch',
  function(e) {
    e.respondWith(fetch(e.request));
  }
};
